# 위협 모델(STRIDE 예시)

자산
- 사용자 데이터, 인증 토큰, 로그, 구성 파일

행위자
- 합법 사용자, 내부자, 외부 공격자(봇/스크립트), 서드파티 시스템

공격 벡터 및 분류(STRIDE)
- S(위조): 약한 인증 흐름 악용
- T(변조): 입력 검증 미비로 인한 파라미터 변조
- R(부인 방지): 로깅/추적성 부족
- I(정보노출): 에러 메시지에 민감 정보 포함
- D(서비스 거부): 비효율 쿼리로 인한 리소스 소진
- E(권한 상승): 수평/수직 권한 검증 누락

우선순위 및 완화
- 고: 입력 검증/표준 에러 모델 도입, 속도 제한, 인증 토큰 보호
- 중: 의존성 취약점 업데이트, 보안 헤더 설정
- 저: 로깅/모니터링 보강, 운영 경고 구성

테스트 제안
- 유효성 검사 단위/통합 테스트, 보안 헤더 존재 테스트, 취약 경로 페이로드 테스트
