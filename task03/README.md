## 소개

가위, 바위, 보 게임은 프로그래밍 기초를 학습하기에 좋은 프로젝트입니다. 이 프로젝트를 통해 조건문, 사용자 입력, 랜덤 값 생성, 간단한 로직 구현 등 프로그래밍의 기본 개념을 실습할 수 있습니다. Junie는 이러한 프로젝트를 빠르고 효율적으로 생성할 수 있도록 도와주는 AI 코딩 어시스턴트입니다.

## 시작하기 전에

프로젝트를 시작하기 전에 다음 사항을 확인하세요:

1. IntelliJ IDEA가 설치되어 있어야 합니다.
2. Junie 플러그인이 설치 및 활성화되어 있어야 합니다.
3. 기본적인 Java 프로그래밍 지식이 있으면 더 효과적으로 활용할 수 있습니다.

## 프로젝트 생성 단계

### 1. 새 프로젝트 생성

1. IntelliJ IDEA를 실행합니다.
2. `File` > `New` > `Project`를 선택합니다.
3. 왼쪽 패널에서 `Java`를 선택합니다.
4. 프로젝트 이름을 "RockPaperScissors"로 지정합니다.
5. 프로젝트 위치를 선택하고 `Create` 버튼을 클릭합니다.

### 2. Junie에게 프로젝트 생성 요청하기

1. 프로젝트가 생성되면, 기본 소스 디렉토리(일반적으로 `src`)를 마우스 오른쪽 버튼으로 클릭합니다.
2. 컨텍스트 메뉴에서 `New` > `Java Class`를 선택합니다.
3. 클래스 이름을 "RockPaperScissorsGame"으로 입력하고 `OK`를 클릭합니다.
4. 새로 생성된 빈 클래스 파일에서 다음 중 한 가지 방법으로 Junie를 활성화합니다:
   - 단축키 `Alt+J`(Windows/Linux) 또는 `Option+J`(macOS)를 누릅니다.
   - 편집기 내에서 마우스 오른쪽 버튼을 클릭하고 `Junie` > `Ask Junie`를 선택합니다.
   - 상단 메뉴에서 `Tools` > `Junie` > `Ask Junie`를 선택합니다.

5. Junie 대화 창이 나타나면 다음 프롬프트를 입력합니다:

```
가위 바위 보 프로젝트를 만들어줘
```

6. `Send` 버튼을 클릭하거나 `Enter` 키를 누릅니다.

### 3. Junie 응답 검토 및 적용

1. Junie는 가위, 바위, 보 게임의 코드를 생성하여 제안할 것입니다.
2. 제안된 코드를 검토합니다.
3. 코드가 만족스럽다면 `Apply` 버튼을 클릭하여 생성된 코드를 현재 파일에 적용합니다.
4. 필요한 경우 추가 클래스 파일이 필요하다는 안내가 표시될 수 있습니다. 이 경우 안내에 따라 새 파일을 생성하고 코드를 적용합니다.

### 4. 프로젝트 실행 및 테스트

1. 프로젝트 코드가 모두 적용되면, 메인 클래스(main 메소드가 있는 클래스)에서 `Run` 버튼(녹색 삼각형)을 클릭하거나 단축키 `Shift+F10`(Windows/Linux) 또는 `Control+R`(macOS)를 누릅니다.
2. 콘솔 창에서 게임을 플레이하며 정상 작동하는지 확인합니다.
3. 필요한 경우 Junie에게 추가 기능 구현이나 버그 수정을 요청할 수 있습니다.

## Junie를 통해 얻을 수 있는 이점

Junie를 사용하여 가위, 바위, 보 프로젝트를 생성함으로써 다음과 같은 이점을 얻을 수 있습니다:

### 1. 코드 작성 시간 단축

Junie는 기본적인 게임 로직, 사용자 입력 처리, 결과 판정 등을 포함한 전체 프로젝트 코드를 빠르게 생성해줍니다. 이로 인해 직접 코딩하는 것보다 시간을 크게 절약할 수 있습니다.

### 2. 코드 구조 및 설계 학습

Junie가 생성한 코드는 객체 지향 설계 원칙에 따라 구조화되어 있습니다. 이를 통해 적절한 클래스 분리, 메소드 설계, 변수 명명 규칙 등 좋은 코딩 관행을 배울 수 있습니다.

### 3. 코드 이해 및 설명

생성된 코드에 대해 Junie에게 추가 설명을 요청할 수 있습니다. 특정 로직이나 구현 방식이 이해되지 않을 때 Junie에게 물어보면 상세한 설명을 제공받을 수 있습니다.

### 4. 다양한 구현 방식 탐색

같은 요구사항에 대해 다양한 방식으로 구현을 요청할 수 있습니다. 예를 들어, 콘솔 기반, GUI 기반, 또는 웹 기반으로 다양하게 요청해보며 각 접근 방식의 차이점을 학습할 수 있습니다.

### 5. 코드 개선 및 리팩토링

초기 버전의 코드를 생성한 후, Junie에게 코드 개선이나 특정 부분의 리팩토링을 요청할 수 있습니다. 이를 통해 점진적으로 코드 품질을 향상시키는 방법을 배울 수 있습니다.

## 프로젝트 확장하기

기본적인 가위, 바위, 보 게임을 구현한 후, Junie에게 다음과 같은 추가 기능을 요청하여 프로젝트를 확장해볼 수 있습니다:

### 1. 게임 기록 저장 기능

```
가위 바위 보 게임에 게임 결과를 저장하고 조회하는 기능을 추가해줘
```

### 2. 그래픽 사용자 인터페이스(GUI) 추가

```
가위 바위 보 게임에 Swing이나 JavaFX를 사용한 GUI를 추가해줘
```

### 3. 게임 규칙 확장

```
가위 바위 보 게임을 확장해서 가위, 바위, 보, 도마뱀, 스팍 게임으로 만들어줘
```

### 4. 네트워크 기능 추가

```
두 플레이어가 네트워크로 연결해서 플레이할 수 있는 가위 바위 보 게임을 만들어줘
```

### 5. 인공지능 상대 개선

```
가위 바위 보 게임에서 플레이어의 패턴을 학습하는 고급 AI 상대를 추가해줘
```
