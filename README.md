# Junie IntelliJ 프로젝트 가이드

이 프로젝트는 IntelliJ IDEA에서 Junie 플러그인을 활용하여 다양한 개발 작업을 수행하는 방법을 학습하기 위한 가이드입니다. Task 01부터 Task 10까지 단계별로 구성되어 있으며, 각 task는 점진적으로 난이도가 증가하는 구조로 되어 있습니다.



## 프로젝트 구성

### Task 01: IntelliJ IDEA에서 Junie 플러그인 사용 가이드

Junie 플러그인을 IntelliJ IDEA에 설치하고 활성화하는 방법을 학습합니다.

- **주요 내용**: Junie 플러그인 소개, 설치 방법, 활성화 방법, 설정 접근 방법, 문제 해결
- **학습 목표**: Junie 플러그인의 기본 사용법을 익히고 개발 환경을 구성할 수 있습니다.

### Task 02: guideline.md 생성 가이드

프로젝트에 필요한 guideline.md 파일을 생성하는 방법을 학습합니다.

- **주요 내용**: guideline.md 개요, 사전 준비, 생성 방법 (일반 생성 방법, Mastering Junie를 통한 생성 방법), 문제 해결
- **학습 목표**: 프로젝트별 개발 가이드라인을 설정하고 Junie가 이를 활용하여 일관된 코드를 생성하도록 구성할 수 있습니다.

### Task 03: Junie를 통한 가위, 바위, 보 프로젝트 만들기

Junie를 활용하여 간단한 가위, 바위, 보 게임 프로젝트를 생성하는 방법을 학습합니다.

- **주요 내용**: 프로젝트 소개, 시작하기 전에, 프로젝트 생성 단계, Junie를 통해 얻을 수 있는 이점, 프로젝트 확장하기
- **학습 목표**: Junie의 Code 모드를 활용하여 기본적인 Java 프로젝트를 생성하고, 프로그래밍 기초 개념을 실습할 수 있습니다.

### Task 04: 고급 프로젝트 개발 가이드 - 도서관 관리 시스템 구축

MVC 패턴, JUnit 테스트, JDBC, 데이터베이스 연결 등을 활용하여 도서관 관리 시스템을 구축하는 방법을 학습합니다.

- **주요 내용**: 
  - MVC 아키텍처 패턴 적용
  - 도서 관리, 회원 관리, 대출 및 반납 처리, 연체 관리, 도서 예약 시스템 구현
  - JUnit 테스트 작성 및 통합 테스트
  - 데이터베이스 스키마 설계 및 JDBC 연동
- **학습 목표**: 실제 운영 환경에서 활용 가능한 수준의 애플리케이션을 개발하고, 체계적인 프로젝트 구조를 설계할 수 있습니다.

### Task 05: Junie와 MCP 연동 가이드

Junie 설정에서 MCP(Model Checkpoint Protocol)를 연동하는 방법을 학습합니다.

- **주요 내용**: MCP 개요, 프로젝트 구조 설정, .junie 디렉토리 설정, mcp.json 파일 구성, GitHub MCP 연동, 연동 테스트 및 확인
- **학습 목표**: AI 모델 체크포인트를 관리하고 버전 제어하며, GitHub와 연동하여 협업 작업을 지원할 수 있습니다.

### Task 06: 레거시 모듈 리팩터링 & 회귀 테스트 자동화

복잡한 스파게티 코드를 모듈화하고 회귀 방지 테스트를 자동화하는 방법을 학습합니다.

- **주요 내용**: 
  - 레거시 코드 분석 및 의존성 다이어그램 작성
  - 인터페이스 분리 및 추상화 도입
  - Golden Test를 통한 기존 동작 스냅샷 확보
  - 단계적 리팩터링 및 호환성 레이어 제공
  - CI 구성 및 회귀 테스트 자동화
- **학습 목표**: 테스트 커버리지 80% 이상 달성, 순환 의존성 제거, 성능 회귀 방지를 통해 유지보수 가능한 코드로 개선할 수 있습니다.

### Task 07: Contract-first API (서버/클라이언트/계약 테스트)

OpenAPI 스펙을 기준으로 서버 스텁, 타입 안전 SDK, 계약 테스트를 구축하는 방법을 학습합니다.

- **주요 내용**: 
  - OpenAPI 스펙 작성 및 관리
  - 서버 스텁 생성
  - 타입 안전 SDK 구현
  - 계약 테스트 작성 및 CI 통합
  - 스펙과 구현 간 drift 방지
- **학습 목표**: API 스펙을 먼저 정의하고, 이를 기반으로 서버와 클라이언트를 개발하여 타입 안전성과 호환성을 보장할 수 있습니다.

### Task 08: 성능 병목 탐지, 프로파일링, 알고리즘 개선

핫 패스를 식별하고 알고리즘 및 자료구조를 개선하여 성능을 최적화하는 방법을 학습합니다.

- **주요 내용**: 
  - 프로파일링을 통한 성능 병목 탐지
  - 마이크로벤치마크 작성 및 실행
  - 알고리즘 및 자료구조 개선
  - 로드 테스트 스크립트 작성
  - 전/후 성능 비교 보고서 작성
- **학습 목표**: p95/p99 레이턴시 개선, 메모리 피크 감소, 재현 가능한 벤치마크를 통해 성능을 체계적으로 개선할 수 있습니다.

### Task 09: 보안 하드닝 (위협 모델링, 정적/동적 분석, 시크릿 스캐닝)

공격 표면을 정리하고 취약점을 점검 및 완화하며 DevSecOps 체인을 구성하는 방법을 학습합니다.

- **주요 내용**: 
  - STRIDE 기반 위협 모델링
  - SAST/DAST 정적/동적 분석
  - 의존성 취약점 감사 및 업데이트
  - 시크릿 스캐닝 및 유출 방지 정책 적용
  - CI 보안 파이프라인 구성
  - Pre-commit 훅을 통한 보안 검사
- **학습 목표**: 입력 검증 강화, 취약 라이브러리 업데이트, 시크릿 유출 방지를 통해 보안이 강화된 애플리케이션을 개발할 수 있습니다.

### Task 10: E2E 기능 제공 (데이터모델→백엔드→프론트→관측성→릴리즈)

현실적인 신규 기능을 설계부터 배포까지 End-to-End로 제공하는 방법을 학습합니다.

- **주요 내용**: 
  - Epic→Story→Task 분해 및 일정 수립
  - 데이터 모델 설계 및 마이그레이션 정의
  - 백엔드 및 프론트엔드 구현
  - 관측성(Observability) 구성 (로깅, 메트릭, 트레이싱)
  - 단위/통합/E2E 테스트 작성
  - 대시보드 및 알림 구성
  - 장애 없는 마이그레이션 및 롤백 절차 검증
  - 품질 기준(SLO) 정의 및 준수
  - 릴리즈 노트 작성 및 배포
- **학습 목표**: 데이터 모델부터 백엔드, 프론트엔드, 관측성, 릴리즈까지 전체 개발 생명주기를 경험하고, 프로덕션 수준의 기능을 제공할 수 있습니다.

## 사용 방법

각 task 디렉토리에는 해당 작업에 대한 상세한 README.md 파일이 포함되어 있습니다. 각 task의 README.md를 참고하여 단계별로 학습하시기 바랍니다.

## 필수 요구사항

- IntelliJ IDEA (최신 버전 권장)
- Junie 플러그인 설치 및 활성화
- JDK 11 이상
- Git (일부 task에서 필요)
- 기본적인 Java 프로그래밍 지식

## 학습 순서 권장 사항

1. **초급 (Task 01~03)**: Junie 플러그인 기본 사용법과 간단한 프로젝트 생성
2. **중급 (Task 04~05)**: 아키텍처 패턴 적용 및 외부 도구 연동
3. **고급 (Task 06~10)**: 리팩터링, API 설계, 성능 최적화, 보안, E2E 개발

각 task는 독립적으로 학습할 수 있지만, 순서대로 진행하시면 더 효과적으로 학습할 수 있습니다.
