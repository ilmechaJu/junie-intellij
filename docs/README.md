# Docs Index

이 디렉토리는 프로젝트 전반의 공통 문서와 템플릿을 보관합니다. 각 문서는 Task 06~10 플레이북과 연동되어 팀이 일관된 방식으로 산출물을 생성하도록 돕습니다.

## 빠른 시작
1. 작업을 시작하기 전에 루트 `junie-task06-10-playbook.md`를 먼저 읽고 선택한 Task 섹션의 체크리스트를 확정합니다.
2. 아래 템플릿을 복사해 PR에 포함시키고, CI에서 문서와 코드의 동기화를 검증합니다.
3. 변경 영향/마이그레이션이 있는 경우, 반드시 Impact/Migration 문서를 함께 작성하세요.

## 템플릿 개요 및 사용 시점

### 1) ADR 템플릿
- 경로: `docs/adr/ADR-YYYYMMDD-template.md`
- 언제: 아키텍처/기술 의사결정이 필요한 모든 변경(프레임워크 채택, 데이터 모델 변경, 보안 정책 등)
- 산출: 결정 동인(Drivers), 고려 옵션 비교 표, 리스크/완화, 마이그레이션/롤백 요약, 참고 링크
- 팁: PR에 해당 ADR을 링크하고, 상태를 Proposed→Accepted로 업데이트

### 2) 변경 영향(Impact) 템플릿
- 경로: `docs/impact/IMPACT-task-template.md`
- 언제: 사용자 영향, API 변경, 성능/비용/보안 영향이 발생할 수 있는 변경
- 산출: API/계약 변화 표, 데이터/스키마 영향, 성능 가설과 측정 계획, 보안 영향, 롤백/검증 계획
- 팁: 릴리즈 노트 초안의 기반으로 사용

### 3) 마이그레이션(MIG) 템플릿
- 경로: `docs/migrations/MIG-template.md`
- 언제: DB 스키마/데이터/코드 동시 변경, 피처 플래그를 통한 단계적 전환 필요 시
- 산출: 사전 조건, 실행 단계(준비→스키마→백필→릴리즈→검증→클린업), 롤백 계획, 관측성 지표, 체크리스트
- 팁: 샌드박스 리허설 로그/결과 캡처 경로를 문서에 남기기

## 관련 가이드
- 루트 플레이북: `../junie-task06-10-playbook.md`
- 각 Task별 상세 가이드는 `/task06` ~ `/task10`의 README를 참고

## 운영 규칙 체크
- PR 열 때: ADR 1건 이상(해당 시), Impact/Migration 동반, 테스트/CI 통과 스크린샷 또는 링크
- 릴리즈: 태그(v0X.0.0)와 릴리즈 노트(사용자 영향, 마이그레이션, 롤백) 첨부

## 개선 제안
문서 템플릿에 보완이 필요하면 이 폴더에서 PR을 생성하고, 변경 이유를 ADR에 기록해 주세요.
